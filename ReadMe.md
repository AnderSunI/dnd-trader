# D&D Trader — Помощник ГМ

> Тёмный fantasy web-инструмент для D&D/BG3-подобных кампаний: торговцы, предметы, инвентарь, энциклопедия, карта, заметки, лист персонажа и инструменты мастера.

**Текущая тестовая версия:**  
https://dnd-trader.onrender.com

> Проект находится в активной разработке. Это не готовый коммерческий продукт и не публичный open-source-шаблон для перепродажи. Код и материалы опубликованы для демонстрации, разработки, портфолио и некоммерческого использования с указанием автора.

---

## Что это такое

**D&D Trader** — это веб-приложение-помощник для мастера настольной RPG-кампании.

Изначально проект начинался как генератор торговцев для D&D: кузнецы, алхимики, трактирщики, редкие товары, фильтры, цены, корзина и инвентарь игрока. Сейчас проект развивается в более крупную экосистему для ведения кампании:

- торговцы и их товары;
- корзина, покупки и инвентарь;
- личный кабинет игрока;
- энциклопедия D&D/BG3-стиля;
- монстры, предметы, заклинания, классы, расы, боги, лор, состояния, механики;
- карта и заметки;
- LSS / лист персонажа;
- Master Room / инструменты мастера;
- будущий боевой модуль и управление партией.

Главная идея проекта — сделать не сухую таблицу, а удобный RPG-интерфейс, похожий на игровые инвентари и справочники, но пригодный для настольной игры.

---

## Текущий статус

Проект живой и активно меняется. Архитектура постепенно приводится в порядок, старые монолитные части разделяются на модули.

На текущем этапе есть:

- FastAPI backend;
- frontend на HTML/CSS/Vanilla JS;
- PostgreSQL;
- Docker / Docker Compose;
- модуль торговцев;
- модуль инвентаря;
- личный кабинет;
- энциклопедия;
- парсеры и нормализаторы данных;
- модульная CSS-инфраструктура для постепенного разделения большого `styles.css`.

## Основные модули

### Торговцы

Генерация и отображение торговцев с фильтрами:

- тип торговца;
- регион;
- уровень игрока;
- репутация;
- поиск по товарам;
- цена;
- редкость;
- категория;
- магические / обычные предметы;
- сортировка.

У торговцев могут быть разные товары, репутация, описание, регион и профессиональный стиль.

### Инвентарь и корзина

Игрок может просматривать товары, добавлять их в корзину, оформлять покупки и видеть свой инвентарь.

Цель модуля — приблизить поведение к RPG-инвентарю: карточки предметов, редкости, стоимость, свойства, описание и механика.

### Энциклопедия

Энциклопедия — отдельный большой модуль справочника. Она нужна не только игрокам, но и мастеру.

Внутри могут быть:

- монстры;
- предметы;
- заклинания;
- классы;
- расы;
- боги;
- фракции;
- состояния;
- механики;
- лор;
- локации.

Подход к данным: лучше сохранить сырой/грязный текст и пометить его для проверки, чем потерять механику. Поэтому часть данных хранится в raw-first формате: сначала сохраняется оригинальный источник, потом нормализованная версия.

### LSS / лист персонажа

Модуль персонажа должен стать удобной карточкой героя: характеристики, навыки, заметки, инвентарь, заклинания, связи с Master Room и боевым модулем.

Название LSS временное и в будущем может быть заменено на собственное название проекта.

### Master Room

Зона мастера для управления кампанией, игроками, событиями, доступами, торговцами, предметами и будущими боевыми сценами.

## Технологии

- **Backend:** Python + FastAPI
- **Database:** PostgreSQL
- **Frontend:** HTML, CSS, Vanilla JavaScript
- **Containerization:** Docker, Docker Compose
- **Deployment:** Render
- **Tools:** Git, curl, Linux, shell scripts
- **Data pipeline:** парсеры, нормализация JSON, raw/normalized/canonical подход

---

## Архитектурный подход

Проект развивается не как одноразовый сайт, а как большая RPG-система.

Основные принципы:
**CSS постепенно делится на модули.**  
   Старый `styles.css` остаётся базовой несущей стеной, а новые модули подключаются отдельно:
   - `bestiary-classes.css`
   - `bestiary-monsters.css`
   - `bestiary-items.css`
   - `traders.css`
   - `inventory.css`
   - и так далее.

## Установка и запуск локально

### 1. Клонировать репозиторий

```bash
git clone https://github.com/AnderSunI/dnd-trader.git
cd dnd-trader
```

### 2. Установить Docker и Docker Compose

Для Ubuntu:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y docker.io docker-compose
sudo usermod -aG docker $USER
newgrp docker
```

### 3. Запустить проект

```bash
docker compose up --build
```

или, если в проекте используется старая команда:

```bash
docker-compose up --build
```

### 4. Открыть локально

Обычно проект доступен по адресу:

```text
http://127.0.0.1:8000
```

## Проверка после изменений

Перед коммитом желательно проверить:

```bash
git status --short
python3 -m py_compile app/main.py
node --check frontend/js/bestiari.js
```

Для CSS-модулей:

```bash
curl -sS -o /tmp/check.css -w "%{http_code} %{content_type}\n" http://127.0.0.1:8000/modules/bestiary-classes.css
```

Ожидаемый результат:

```text
200 text/css
```

## Разработка CSS

Сейчас проект переходит от одного большого `frontend/css/styles.css` к модульной структуре.

Важно:

- `styles.css` пока остаётся основным CSS-файлом;
- новые CSS-модули подключаются после него;
- нельзя заменять `styles.css` на пустой import-entry без проверки backend-статики;
- каждый новый модуль должен иметь понятную шапку: за что отвечает и что не трогает.

Пример подключения:

```html
<link rel="stylesheet" href="/styles.css" />
<link rel="stylesheet" href="/modules/bestiary-classes.css" />
<link rel="stylesheet" href="/modules/bestiary-monsters.css" />
```

## Данные и источники

Проект использует данные из разных открытых и фанатских источников по D&D/BG3-тематике. Данные проходят этапы очистки и нормализации.

Важное правило:

> Сырые данные не удаляются без необходимости. Если запись грязная или неполная, она должна быть сохранена и помечена для проверки.

Это особенно важно для:

- предметов;
- заклинаний;
- монстров;
- легендарных действий;
- условий;
- ссылок на источники;
- вариантов из разных редакций/источников.
