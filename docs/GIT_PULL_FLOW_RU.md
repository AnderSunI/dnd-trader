# Как забираются изменения из git (коротко)

## Что получишь при `git pull`
Если ты делаешь обычный `git pull` целевой ветки, ты получаешь **последний срез**,
но внутри него находятся **все коммиты последовательно** (вся история до текущей точки).

То есть:
- да, у тебя будет актуальное последнее состояние,
- и да, при этом сохраняются все промежуточные коммиты в истории.

## Если нужен не весь хвост, а только часть
Можно брать выборочно:
- `git log --oneline` — посмотреть цепочку,
- `git cherry-pick <commit>` — взять конкретный коммит,
- `git checkout <commit>` — посмотреть проект на конкретной точке времени.

## Практика для нашего потока
1. Делаем маленькие коммиты (один шаг = один коммит).
2. После checkpoint ты делаешь `git pull`.
3. Если нужно откатить/проверить этап — смотришь конкретный commit из `git log`.
