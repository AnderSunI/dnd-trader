# AGENTS.md

## D&D Trader local rules

Работаем только ЛОКАЛЬНО в этом проекте.
Базовая ветка для сравнения и понимания контекста: develop.
Текущая рабочая ветка — локальная work-ветка пользователя.
Никогда не коммить и не пушь без отдельного подтверждения.

## Main working style
- Сначала анализ проекта, потом краткий план, потом код.
- Перед любым большим изменением сначала показать:
  - какие файлы будут изменены
  - почему именно они
- Работать по возможности по одному файлу за раз.
- Если меняется файл — отдавать полный финальный файл целиком.
- Не делать пустые mock-заглушки вместо рабочего кода.
- Не выдумывать архитектуру мимо существующего проекта.
- Не плодить лишние мелкие файлы без реальной необходимости.

## Data safety rules
- Ничего не терять из lore, mechanics_text, grants, raw, unclassified, notes.
- Лучше сохранить грязные данные, чем потерять важные.
- Cleanup только поверх сохранённого слоя, а не вместо него.

## Branch and Git rules
- Не трогать main напрямую.
- Не пушить в develop без подтверждения.
- Работать только в текущей локальной work-ветке пользователя.
- Перед изменениями проверять текущую ветку и git status.
- Если есть незакоммиченные изменения — сначала предупредить.

## Startup ritual for every new/restarted session
В начале каждой новой сессии:
1. Определи текущую ветку.
2. Прочитай AGENTS.md.
3. Посмотри git status.
4. Коротко скажи:
   - на какой ветке находимся
   - чистое ли рабочее дерево
   - на чём, по твоему мнению, мы остановились
   - какой следующий минимальный шаг логичен
5. Ничего не меняй, пока не покажешь этот краткий стартовый статус.

## Project priorities right now
Текущий приоритет зависит от запроса пользователя в диалоге.
Если запрос про GM-модуль:
- сначала проверить текущий vertical slice
- не переписывать полпроекта
- выбрать минимальный file-level шаг
Если запрос про items/BG3:
- сначала safe/preserved/master logic
- не резать ability/mechanics pages как мусор

## What “done” means
Задача считается сделанной только если:
- есть краткое объяснение что изменено
- есть список изменённых файлов
- есть способ протестировать сценарий end-to-end
- нет скрытых архитектурных сюрпризов без предупреждения
